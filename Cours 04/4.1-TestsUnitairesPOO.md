# Introduction aux tests unitaires en POO Java

## 1. Pourquoi faire des tests unitaires ?

Les tests unitaires servent à :

- Vérifier que le code fonctionne correctement\
- Prévenir les régressions lors des modifications\
- Documenter le comportement attendu d'une classe\
- Faciliter la maintenance du code\
- Développer avec plus de confiance

## 2. Qu'est-ce qu'un test unitaire ?

Un **test unitaire** valide une petite unité de code :

- Une méthode
- Une classe
- Un comportement précis

On teste **UNE** chose à la fois.

## 3. Principe AAA (Arrange / Act / Assert)

C'est la structure standard d'un bon test unitaire.

### 1️⃣ Arrange (Préparer)

On crée les objets et on prépare le contexte.

Exemples :

- Créer un objet
- Définir des valeurs initiales
- Préparer des données

``` java
Tracteur t = new Tracteur("John Deere");
```

### 2️⃣ Act (Agir)

On exécute l'action à tester.

``` java
t.rouler();
```

### 3️⃣ Assert (Vérifier)

On vérifie que le résultat est celui attendu.

``` java
assertEquals(10, t.getCarburant());
```

👉 Si la valeur est correcte → test réussi\
👉 Sinon → test échoue

## 4. 🚜 Exemple simple : Classe Tracteur

### Classe à tester

``` java
public class Tracteur {

    private String marque;
    private int carburant;

    public Tracteur(String marque) {
        this.marque = marque;
        this.carburant = 20;
    }

    public void rouler() {
        if(carburant > 0) {
            carburant -= 5;
        }
    }

    public void faireLePlein() {
        carburant = 20;
    }

    public int getCarburant() {
        return carburant;
    }

    public String getMarque() {
        return marque;
    }
}
```

## 5. Exemple de test unitaire (JUnit)

``` java
import org.junit.Test;
import static org.junit.Assert.*;

public class TracteurTest {

    @Test
    public void testRouler() {

        // Arrange
        Tracteur t = new Tracteur("John Deere");

        // Act
        t.rouler();

        // Assert
        assertEquals(15, t.getCarburant());
    }
}
```

## 6. Bonnes pratiques

- Un test = un comportement
- Nom de test clair
- AAA visible dans le code
- Tests simples et courts
- Tester les cas normaux ET limites

## 7. Ce qu'il faut éviter

- Plusieurs actions dans un test
- Tests dépendants entre eux
- Tests trop complexes
- Assertions absentes

## 8. Les assertions (asserts) en tests unitaires Java

### À quoi servent les asserts ?

Un `assert` est l'instruction qui permet de **vérifier qu'un résultat
correspond à ce qui est attendu** dans un test unitaire.

Les assertions servent à :

- comparer une valeur obtenue avec une valeur attendue\
- vérifier qu'un objet est nul ou non\
- confirmer qu'une exception est lancée\
- valider des conditions logiques (vrai/faux)\
- comparer des chaînes, nombres et objets

**Sans assertion, un test ne valide rien.**

### Principe de base

Un assert répond toujours à la question :

> *Le résultat correspond-il à ce que j'attendais ?*

Exemple :

``` java
assertEquals(5, calculerSomme(2,3));
```

✔️ Test réussi si résultat = 5\
❌ Test échoue sinon

## 9. Syntaxe générale (JUnit)

``` java
assertEquals(valeurAttendue, valeurObtenue);
```

## 10. Asserts les plus utilisés

### Asserts pour les String

|Assert|Exemple|Explication|
|---|---|---|
|`assertEquals`|`assertEquals("chat", nom);`|Vérifie l'égalité exacte|
|`assertNotEquals`|`assertNotEquals("chien", nom);`|Vérifie la différence|
|`assertTrue`|`assertTrue(texte.contains("abc"));`|Condition vraie|
|`assertFalse`|`assertFalse(texte.isEmpty());`|Condition fausse|
|`assertNull`|`assertNull(resultat);`|Doit être null|
|`assertNotNull`|`assertNotNull(nom);`|Ne doit pas être null|

### Asserts pour les int

|Assert|Exemple|Explication|
|---|---|---|
|`assertEquals`|`assertEquals(10, nb);`|Valeur exacte|
|`assertNotEquals`|`assertNotEquals(0, nb);`|Valeur différente|
|`assertTrue`|`assertTrue(nb > 0);`|Condition vraie|
|`assertFalse`|`assertFalse(nb < 0);`|Condition fausse|

### Asserts pour les double

|Assert|Exemple|Explication|
|---|---|---|
|`assertEquals`|(avec `assertEquals(3.14, pi, 0.001);`|Tolérance d'erreur delta)|
|`assertNotEquals`|`assertNotEquals(2.0, valeur, 0.01);`|Différence tolérée|
|`assertTrue`|`assertTrue(valeur > 0.0);`|Condition vraie|

## 11. Erreurs fréquentes chez les débutants/stagiaires

- Oublier les assertions
- Tester plusieurs choses dans un seul test
- Comparer des doubles sans delta
- Tester une logique trop complexe à la fois

## 12. Résumé

Les tests unitaires :

- sécurisent le code
- facilitent la maintenance
- encouragent de bonnes pratiques POO

La structure AAA aide à écrire des tests clairs et efficaces.
