# `Comparable<T>` en Java — l'essentiel

## 1. Objectif de la fiche

- Définir l'**ordre naturel** d'une classe pour pouvoir **trier une `ArrayList`** simplement :

  ```java
  Collections.sort(liste);      // Java 7-
  liste.sort(null);             // Java 8+ (null => ordre naturel)
  ```

---

## 2. Aperçu rapide – Interface `Comparable`

> **Référence** :  
[Interface `Comparable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html)

```java
// Déclaration de l'interface.
public interface Comparable<T> {
    public int compareTo(T other);
}
```

L'interface `Comparable` de Java propose une seule méthode, `compareTo()`, qui sert à **déterminer l'ordre naturel des objets d'une classe**. Elle permet ainsi de **trier les objets**.

La méthode respecte le principe suivant : `compareTo` retourne un entier

- **`< 0`** si `this` est **avant** `other`
- **`0`** si **égalité d'ordre**
- **`> 0`** si `this` est **après** `other`

### Pour qu'une classe implémente `Comparable`, il faut 3 choses :

1. Ajouter la mention `implements Comparable<T>` avant la `{` de la classe
2. Implémenter la méthode `compareTo` qui définit l'ordre naturel des objets selon les critères établis
3. Générer la méthode `equals()` [nous utiliserons un modèle simple]

## 3. Contrat minimal à respecter

La méthode `compareTo` doit s'assurer de respecter les propriétés suivantes :

- **Antisymétrie** : `sgn(a.compareTo(b)) == -sgn(b.compareTo(a))`
- **Transitivité** : si `a > b` et `b > c` alors `a > c`
- **Cohérence attendue avec `equals`** : si `compareTo` retourne `0`, les objets *devraient* être égaux selon la logique métier (voir §6).

## 4. Bonnes pratiques pour `compareTo`

- Utiliser les méthodes sûres : `Integer.compare`, `Long.compare`, `Double.compare`, etc. pour les types numériques.
- Chaîner les critères (tie-breakers) proprement.
- Décider **explicitement** si les `null` sont autorisés (sinon, documenter que non).
- Documenter le **sens** de l'ordre (par défaut : croissant).

## 5. `Comparable` vs `Comparator` (pour situer)

- **`Comparable`** : ordre « naturel » **intrinsèque** à la classe (défini une fois pour toutes).
- **`Comparator`** : ordres **alternatifs** (non traités ici, plus tard dans le cours).

## 6. Remarque importante sur `.equals()`

Si **`compareTo` retourne `0` pour deux objets** (même ordre), ils devraient **en général** être considérés **égaux** (même identité logique).

Deux options cohérentes sont possibles :

  1. **Même identité pour les deux** : alors `equals` retourne `true` et utilise les **mêmes** champs que ceux qui peuvent mener à `compareTo == 0`.
  2. **Identité différente** : documenter clairement que `compareTo == 0` **n'implique pas** l'égalité *objet*, c'est à dire `equals` peut retourner `false` (cas rares à privilégier seulement si nécessaire).

## 7. Exemple minimal et propre

### Classe `Etudiant` — ordre naturel : `nom` (insensible à la casse), puis `prenom`, puis `matricule`

```java
import java.util.Objects;

public final class Etudiant implements Comparable<Etudiant> {
    private final String matricule; // ex: "20261234"
    private final String nom;
    private final String prenom;
    private final double moyenne;   // non utilisé dans l'ordre naturel

    public Etudiant(String matricule, String nom, String prenom, double moyenne) {
        this.matricule = Objects.requireNonNull(matricule, "matricule");
        this.nom = Objects.requireNonNull(nom, "nom");
        this.prenom = Objects.requireNonNull(prenom, "prenom");
        this.moyenne = moyenne;
    }

    public String getMatricule() { return matricule; }
    public String getNom() { return nom; }
    public String getPrenom() { return prenom; }
    public double getMoyenne() { return moyenne; }

    /**
     * Ordre naturel:
     * 1) nom (insensible à la casse)
     * 2) prenom (insensible à la casse)
     * 3) matricule (croissant)
     *
     * Remarque: Si compareTo(this, other) == 0, equals(this, other) sera également vrai,
     * car equals repose sur les mêmes champs (nom, prenom, matricule).
     */
    @Override
    public int compareTo(Etudiant other) {
        Objects.requireNonNull(other, "other");
        int byNom = String.CASE_INSENSITIVE_ORDER.compare(this.nom, other.nom);
        if (byNom != 0) return byNom;

        int byPrenom = String.CASE_INSENSITIVE_ORDER.compare(this.prenom, other.prenom);
        if (byPrenom != 0) return byPrenom;

        return this.matricule.compareTo(other.matricule); // String.compareTo est sûr
    }

    /**
     * Aligné avec l'ordre naturel: deux étudiants sont égaux s'ils ont
     * le même nom, le même prénom et le même matricule.
     */
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Etudiant other)) return false;
        return nom.equalsIgnoreCase(other.nom)
            && prenom.equalsIgnoreCase(other.prenom)
            && matricule.equals(other.matricule);
    }

    @Override
    public int hashCode() {
        // Normalise nom et prenom pour refléter equals (insensible à la casse)
        return Objects.hash(nom.toLowerCase(), prenom.toLowerCase(), matricule);
    }

    @Override
    public String toString() {
        return nom + ", " + prenom + " (" + matricule + ") - " + moyenne;
    }
}
```

### Tri d'une `ArrayList`

```java
import java.util.*;

public class DemoTri {
    public static void main(String[] args) {
        List<Etudiant> etudiants = new ArrayList<>();
        etudiants.add(new Etudiant("20261234", "Émond", "Zoé", 84.5));
        etudiants.add(new Etudiant("20264567", "emond", "Alice", 91.0));
        etudiants.add(new Etudiant("20267890", "Bouchard", "Marc", 76.2));

        // Java 8+ : ordre naturel (Comparable)
        etudiants.sort(null);

        // Java 7- : Collections.sort(etudiants);

        etudiants.forEach(System.out::println);
    }
}
```

## Points à retenir

- Utiliser `String.CASE_INSENSITIVE_ORDER` (simple et efficace).
- Chaîner les critères jusqu'à distinguer les objets (éviter les faux « 0 »).
- Garder `equals` **cohérent** avec les critères de `compareTo` qui mènent à `0`.
- Pour les accents et un tri linguistique plus fin : envisager `Collator` (fr-CA) plus tard si besoin.
