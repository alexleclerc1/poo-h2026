
# Cours de POO en Java — Le Polymorphisme (Version Simplifiée)

## 1. Qu’est‑ce que le polymorphisme ?
Le **polymorphisme** permet d’utiliser **une même référence** (ex. `Vehicule`) pour manipuler **plusieurs types d’objets différents** (Moto, Voiture, Tracteur). 

Cela permet d’écrire du **code plus simple**, **plus flexible**, et **moins répétitif**.

Exemple :
```java
Vehicule v = new Moto();
v.klaxon(); // Appelle le klaxon de Moto
```

Même si `v` est de type `Vehicule`, Java utilise **la vraie nature de l’objet** (`Moto`) au moment de l’exécution.

## 2. Override vs Overload (simplifié)
### Redéfinition (Override)
Une sous‑classe **change** le comportement d’une méthode héritée.
- même nom
- mêmes paramètres
- exécution : **au moment où le programme tourne** (liaison dynamique)

Exemple :
```java
class Moto extends Vehicule {
    @Override
    String klaxon() { return "pin pin"; }
}
```

### ➕ Surcharge (Overload)
On crée **plusieurs versions** d’une méthode **avec des paramètres différents**.
- même nom
- paramètres différents
- exécution : **décidée par le compilateur** (liaison statique)

Exemple :
```java
void calculer(int x) {}
void calculer(double x) {}
```

**Résumé rapide**
| Overload | Override |
|----------|----------|
| Plusieurs formes d’une même méthode | Remplacer une méthode héritée |
| Décidé à la compilation | Décidé à l’exécution |
| Paramètres différents | Paramètres identiques |

---

## 3. Liaison statique vs dynamique
### Liaison statique
Décision faite **par le compilateur** avant d’exécuter le programme.
- surcharge (overload)
- méthodes `static`
- méthodes `private`

### Liaison dynamique
Décision faite **pendant l’exécution** selon l’objet réel.
- override (redéfinition)
- appels polymorphes via un type parent

Exemple :
```java
Vehicule v = new Tracteur();
v.klaxon(); // -> version Tracteur (liaison dynamique)
```

## 4. Exemple simple avec Moto, Voiture et Tracteur
On ajoute :
- **odomètre (km)** pour Moto et Voiture
- **horamètre (heures)** pour Tracteur

### Classe abstraite
```java
abstract class Vehicule {
    String marque;
    String modele;
    Vehicule(String m, String mo){ marque=m; modele=mo; }

    abstract String klaxon();
    abstract double vitesseMax();
}
```

### Interfaces simples
```java
interface Kilometrique {
    void rouler(double km);
    double getOdometre();
}

interface Horaire {
    void travailler(double h);
    double getHorametre();
}
```

### Moto
```java
class Moto extends Vehicule implements Kilometrique {
    double odometre = 0;
    Moto(){ super("Honda", "CB500F"); }

    public String klaxon(){ return "pin pin"; }
    public double vitesseMax(){ return 180; }

    public void rouler(double km){ odometre += km; }
    public double getOdometre(){ return odometre; }
}
```

### Voiture
```java
class Voiture extends Vehicule implements Kilometrique {
    double odometre = 0;
    Voiture(){ super("Toyota", "Corolla"); }

    public String klaxon(){ return "pouet pouet"; }
    public double vitesseMax(){ return 210; }

    public void rouler(double km){ odometre += km; }
    public double getOdometre(){ return odometre; }
}
```

### Tracteur
```java
class Tracteur extends Vehicule implements Horaire {
    double horametre = 0;
    Tracteur(){ super("John Deere", "6120M"); }

    public String klaxon(){ return "BRAAAP"; }
    public double vitesseMax(){ return 40; }

    public void travailler(double h){ horametre += h; }
    public double getHorametre(){ return horametre; }
}
```

---

## 5. Démonstration du polymorphisme
```java
List<Vehicule> flotte = List.of(new Moto(), new Voiture(), new Tracteur());

for (Vehicule v : flotte) {
    System.out.println(v.klaxon()); // chacun sa version
}
```

---

## 6. Ce qu’il faut retenir
- Le polymorphisme permet de **manipuler différents objets** avec un **même type parent**.
- L’override est **polymorphe** ; l’overload **ne l’est pas**.
- La liaison dynamique est au cœur de la POO.
- Les interfaces permettent de donner des **capacités différentes** (ex. km vs heures).
