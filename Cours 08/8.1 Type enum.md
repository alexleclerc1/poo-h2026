# Type `enum` en Java

## 1. Qu'est-ce que l'`enum` ?

> **Définition** : L'`enum` **(ou l'énumération)** est une **classe java spéciale** qui représente ***un groupe de constantes*** (Comme des variables `final`)

### Syntaxe

Pour créer une *énumération*, on utilise le mot clé `enum` (là où on aurait mis `class` ou `interface`) suivit du nom de l'énumération et des constantes qu'elle contient. 

> ## Exemple
>
>**TOUTES LES VALEURS DE L'ENUM DOIVENT ÊTRE EN MAJUSCULES**
>```java
>enum Etat {
>    ACTIF,
>    GELÉ,
>    FERMÉ, // <-- Qu'est-ce que fait cette virgule ici !
>}
>```
>On peut accéder à la valeur d'une énumération par le point :
>```java
>Etat monEtat = Etat.ACTIF;
>```

## 2. Cas d'utilisation

### 2.1 Fichier appart

On peut définir un enum dans un fichier appart comme une classe. On peut ensuite importer l'énumération pour l'usage dans une autre classe.

#### `Langue.java`
```java
enum Langue {
    FRANÇAIS,
    ANGLAIS,
    ESPAGNOL,
    ALLEMAND,
    JAPONAIS
}
```

#### `App.java`
```java
public class Main {
    public static void main(String[] args) {

        // On accède à l'enum directement
        // Attention: Que doit-on vérifier si notre enum est dans un package ?
        Langue AppLangue = Langue.ALLEMAND;
    }
}

```


### 2.2 Dans une classe

On peut définir un `enum` à l'intérieur d'une classe pour son usage personnel, ou si l'énumération est `public`, on peut l'utiliser en l'accédant par la classe

*Précision*: le comportement est le même avec le mot clé `static` devant `enum`, **l'enum appartient à la classe et non à une instance.**

#### `Moto.java`
```java
public class Moto {
    // La liste de marque relative à la classe Moto
    public enum Marque {
        YAMAHA,
        SUZUKI,
        KAWASAKI,
        HARLEY
    }

    // L'attribut privé qui enregistre la marque de la moto 
    private Marque marque;

    // Constructeur par défaut
    // On peut utiliser une valeur de base de l'énumration par accès direct.
    public Moto() {
        this.marque = Marque.HARLEY; // Est-ce pertinent d'avoir une harley par défaut ? Quelle solution "par défaut" peut-on faire ?
    }

    // Constructeur surchargé
    // Lors de la création d'une moto, on peut passer une marque en paramètre
    public Moto(Marque marque) {
        this.marque = marque;
    }
}
```

#### `App.java`
```java
public class Main {
    public static void main(String[] args) {

        // On accède à l'enum par Moto.Marque
        Moto maMoto = new Moto(Moto.Marque.KAWASAKI);
    }
}
```

### 2.3 Dans une instruction `switch`

L'énumération est souvent utilisée avec une instruction `switch` pour gérer facilement et précisément des cas.

*Utilité*: Pas besoin de vérifier des cas autres que ceux définis dans l'énumération, contrairement à une chaîne de caractère où celle-ci pourrait présenter un autre cas non défini ou une anomalie d'écriture.

#### `App.java`
```java
enum Etat {
    ACTIF,
    GELÉ,
    FERMÉ
}

public class Main {
    public static void main(String[] args) {

        Etat monEtat = Etat.ACTIF;

        switch(monEtat) {
            case ACTIF: // Question : Pourquoi pas Etat.ACTIF?
                System.out.println("Vous êtes actif.");
                break;
            case GELÉ:
                System.out.println("Vous êtes gelé, veuillez contacter votre administrateur");
                break;
            case FERMÉ:
                System.out.println("Votre compte est fermé.")
                break;
            default:
                System.out.println("État inconnu")
                break;
        }
    }
}
```

### 2.4 Dans une boucle

Le type `enum` comporte une méthode `values()` qui retourne la liste des constantes. On peut alors itérer (faire une boucle) sur la liste des constantes.

```java
for (Marque uneMarque : Marque.values()) {
    System.out.println(uneMarque);
}
```

## 3. Enum comme une classe

L'énumération est en fait une `classe` (comme presque tout en java en fait !). Alors,

> Qu'est-ce qu'on peut mettre dans une classe java ?

### Un constructeur, des attributs et des méthodes

#### *La différence*
- Les constantes des énumérations sont `public`, `static`, et `final`.
- `enum` ne peut pas être utilisé pour créer des objets et ne peut pas étendre d'autres classes (**mais peut implémenter des interfaces**)

#### Exemple

```java
enum Etat {
    ACTIF("Vous êtes actif"),
    GELÉ("Votre compte est gelé, contacter votre administrateur!"),
    FERMÉ("Votre compte est fermé"); // Un point-virgule ?

    // Attribut pour chaque constante
    private String description;

    // Construction pour chaque constante
    private Etat(String description) {
        this.description = description;
    }

    // Méthode accessible pour chaque constante
    public String getDescription() {
        return description
    }
}

public class Main {
  public static void main(String[] args) {

    Etat monEtat = Level.GELÉ;

    // Imprime la description relié à la valeur de l'enum
    System.out.println(monEtat.getDescription());
  }
}
```

# Quand utiliser des `enum` ?

> Utiliser des `enum` quand on a des valeurs constantes qui ne changeront pas dans l'exécution du programme, comme les jours de la semaine, les couleurs des cartes à jouer, des marques, etc.
